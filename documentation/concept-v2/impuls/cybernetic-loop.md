# ◐ IMPULS – Cybernetic Loop

> **Schicht:** 4 – Handlung
> **Sphäre:** ECHO + NOA (System)
> **Typ:** Feedback-Schleife

---

## Konzept

Der **Cybernetic Loop** ist das Herzstück von Erynoa: Ein geschlossener Regelkreis, der Wahrnehmung, Entscheidung, Handlung und Feedback verbindet.

---

## Loop-Phasen

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   CYBERNETIC LOOP PHASES                                                   │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   1. PERCEPTION (ERY)                                               │  │
│   │      ════════════════                                               │  │
│   │      Agent nimmt Umgebung wahr:                                    │  │
│   │      - Discovery: Welche Optionen gibt es?                         │  │
│   │      - Trust: Wem kann ich vertrauen?                              │  │
│   │      - Constraints: Was ist erlaubt?                               │  │
│   │           │                                                         │  │
│   │           ▼                                                         │  │
│   │   2. DECISION (ECHO)                                                │  │
│   │      ═══════════════                                                │  │
│   │      Agent entscheidet basierend auf:                              │  │
│   │      - Policy: Welche Regeln gelten?                               │  │
│   │      - Intent: Was will ich erreichen?                             │  │
│   │      - Negotiation: Welches Angebot nehme ich?                     │  │
│   │           │                                                         │  │
│   │           ▼                                                         │  │
│   │   3. ACTION (ECHO → NOA)                                            │  │
│   │      ════════════════════                                           │  │
│   │      Agent handelt:                                                │  │
│   │      - Transaction: Zahlung, Dienstleistung                        │  │
│   │      - State Change: AMO-Transition                                │  │
│   │      - Commitment: Vereinbarung eingegangen                        │  │
│   │           │                                                         │  │
│   │           ▼                                                         │  │
│   │   4. OBSERVATION (NOA)                                              │  │
│   │      ═════════════════                                              │  │
│   │      System beobachtet Ergebnis:                                   │  │
│   │      - Success/Failure: Hat es funktioniert?                       │  │
│   │      - Metrics: Qualität, Dauer, Kosten                            │  │
│   │      - Evidence: Unveränderlicher Beweis                           │  │
│   │           │                                                         │  │
│   │           ▼                                                         │  │
│   │   5. FEEDBACK (NOA → ERY)                                           │  │
│   │      ═════════════════════                                          │  │
│   │      System lernt:                                                 │  │
│   │      - Trust Update: Reputation anpassen                           │  │
│   │      - Karma Update: Punkte vergeben/abziehen                      │  │
│   │      - Knowledge Update: Neue Informationen                        │  │
│   │           │                                                         │  │
│   │           └──────────────────▶ zurück zu 1. PERCEPTION              │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   BEISPIEL: EV-LADEVORGANG                                                 │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   1. PERCEPTION                                                     │  │
│   │      Fahrzeug-Agent erkennt: Batterie bei 20%                      │  │
│   │      Discovery findet: 5 Ladestationen in 5km Radius               │  │
│   │      Trust-Check: 3 Stationen haben Trust > 0.7                    │  │
│   │                                                                     │  │
│   │   2. DECISION                                                       │  │
│   │      Policy evaluiert Angebote:                                    │  │
│   │      - Station A: 0.42€/kWh, Trust 0.92, 1.2km → Score: 0.94       │  │
│   │      - Station B: 0.38€/kWh, Trust 0.71, 3.5km → Score: 0.81       │  │
│   │      Agent wählt Station A                                         │  │
│   │                                                                     │  │
│   │   3. ACTION                                                         │  │
│   │      Intent → Negotiation → Agreement                              │  │
│   │      Ladevorgang startet                                           │  │
│   │      Streaming Payment beginnt                                     │  │
│   │                                                                     │  │
│   │   4. OBSERVATION                                                    │  │
│   │      Ladevorgang abgeschlossen: 45kWh in 28min                     │  │
│   │      Zahlung: 18.90€                                               │  │
│   │      Qualität: Wie erwartet                                        │  │
│   │      Transaktion finalisiert auf NOA                               │  │
│   │                                                                     │  │
│   │   5. FEEDBACK                                                       │  │
│   │      Station A: +0.02 Reliability                                  │  │
│   │      Fahrzeug: +0.02 Integrity (Zahlung korrekt)                   │  │
│   │      Beide: Karma-Punkte akkumuliert                               │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

## Emergenz durch Feedback

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   EMERGENT BEHAVIOR                                                        │
│                                                                             │
│   Einzelne Loops akkumulieren zu Systemverhalten:                          │
│                                                                             │
│   ┌───────────────────────────────────────────────────────────────────┐    │
│   │                                                                   │    │
│   │   Loop 1: Agent A wählt Provider X                               │    │
│   │   Loop 2: Agent B sieht X's hohen Trust, wählt auch X            │    │
│   │   Loop 3: X erhält mehr Nutzung → mehr Feedback → höherer Trust  │    │
│   │   Loop 4: Mehr Agenten wählen X → Netzwerkeffekt                 │    │
│   │                                                                   │    │
│   │   ABER AUCH:                                                      │    │
│   │                                                                   │    │
│   │   Loop 100: X macht Fehler → negativer Trust-Event               │    │
│   │   Loop 101: Weniger Agenten wählen X                             │    │
│   │   Loop 102: X verbessert Service → Trust erholt sich             │    │
│   │                                                                   │    │
│   │   Das System ist selbstregulierend.                               │    │
│   │                                                                   │    │
│   └───────────────────────────────────────────────────────────────────┘    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
